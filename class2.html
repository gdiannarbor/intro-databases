Pieces and parts of a relational database

A relational database consists of many different elements, some of which include:

  • schema
  • tables
  • views
  • relationships
  • indexes
  • functions
  • stored procedures
  • triggers
  • server objects
  • and other elements

Schema

  • Is a way to logically group database objects
  • Acts as a blueprint for how the database is constructed
  • In a relational database, the schema defines relationships, tables, fields in tables etc.

Tables

In a relational database, a table is an organized set of data using columns and rows.

You may sometimes hear a table referred to as a "relation".

The SELECT Statement

  • We use the SELECT command to show/query data from a database
  • The output from a SELECT statement is always a grid of rows and columns.
  • The most simple SELECT statement retrieves all values from a single table
  • SELECT columns FROM database

Note: SQL is not case sensitive

SELECT * FROM actor;

This will return all of the results from the actor table.

SELECT DISTINCT

DISTINCT keyword is used to return only distinct values, if there are duplicate values

Note: Some database systems require a semi colon after each SQL statement. PostgreSQL is one of them.

SELECT distinct first_name FROM actor;

This will return a distinct list of first names from the actor table in the dvdrentals db

The WHERE Clause

Returns a subset of rows filtered on the WHERE clause criteria

SELECT .. WHERE statement can be made on one line:


Note: PostgreSQL uses double-quotes to look for internal database object. Use single-quote to wrap string values.

SELECT * FROM actor WHERE first_name = 'Bette';

WHERE Clause Operators

Operator Description
= Equal
<> Not equal. Note: In some versions of SQL, this may be written as !=
> Greater than
< Less than
>= Greater than or equal
<= Less than or equal
BETWEEN Between an inclusive range
LIKE Search for a pattern (can use a wildcard character)
IN To specify multiple possible values for a column

Not Equal

SELECT * FROM actor WHERE first_name <> 'Catherine';

Greater Than

SELECT * FROM rental WHERE rental_date > '2005-12-31';

Less Than

SELECT * FROM film WHERE rental_duration < 4;

BETWEEN

SELECT * FROM film WHERE replacement_cost BETWEEN 21.99 AND 24.99;

LIKE

Uses the wildcard character

SELECT * FROM customer WHERE last_name LIKE '%Will%';

Can use the character at the beginning or end of the word/phrase

SELECT * FROM customer WHERE last_name LIKE 'Will%';

IN

SELECT * FROM film WHERE rating IN ('G', 'PG');

Data Manipulation Language (DML)

Used for manipulating data

  • INSERT
  • UPDATE
  • DELETE
  • MERGE

INSERT statement

Used to insert a new row (or set) of data into a table

INSERT statement

INSERT INTO dbo.tableName VALUES x, y, z


              INSERT INTO actor (first_name, last_name)
              VALUES ('Leslie', 'Knope');

Note: If there is an identity or primary key column on a table, you should omit that column from your insert statement.

UPDATE statement

Is used to update or change the value of columns in selected row(s) of data

Use WHERE clause to define your set or row to update

UPDATE example

UPDATE dbo.tableName SET field(s) = new value(s) WHERE condition(s)


              UPDATE film
              SET rental_duration = 4
              WHERE film_id = 692;

DELETE statement

Used to remove data from tables

For UPDATE and DELETE statements, it is helpful to write the WHERE clause first to avoid unintended results

DELETE example

DELETE FROM dbo.tableName WHERE condition(s)


              DELETE FROM payment
              WHERE payment_id = 17873;

Data Definition Language (DDL)

Used for defining data

  • CREATE
  • DROP
  • TRUNCATE
  • ALTER

CREATE statement

  • Can use CREATE to create a database, tables, views, stored procedures, indexes etc. (objects of a db)
  • To create a table, we need to give the table a name and define the columns with a name and datatype
  • Defining primary and foriegn keys as well as constraints and default values is an option here

CREATE example

CREATE TABLE IF NOT EXISTS dbo.tableName
( field definition(s) ); condition(s)


              CREATE TABLE IF NOT EXISTS format
                (ID int,
                type varchar(20),
                download boolean,
                language varchar(30),
                max_number_of_views int,
                updated_at timestamp);

DROP statement

  • Used to delete an object from a database
  • Be very cautious when using this command!
  • Cannot drop a table that does not exist, you will get an error
  • Can use IF EXISTS at the beginning of the statement


              CREATE TABLE IF NOT EXIST sample
                (title varchar(20));
              

              DROP TABLE sample;

TRUNCATE statement

  • Used to delete all the contents of object from a database
  • If all data should be removed from a table, TRUNCATE is a less expensive operation than DELETE
  • Again, be very cautious when using this command!


              INSERT INTO format (type, download, language, max_number_of_views)
              VALUES ('stream', true, 'English', 4);
              

              SELECT * FROM format;
              

              TRUNCATE TABLE format;
              

              SELECT * FROM format;

ALTER statement

  • Used to alter an object in database
  • Not all objects can be altered, this can be vendor specific
  • If an object cannot be altered, you can drop and recreate the object with the changes needed


              ALTER TABLE customer
              ADD birthday date;

Constraints

Limit the type of data that can be inserted into a table


Types include:

  • Primary Key
  • Foreign Key
  • Unique
  • Not NULL
  • Check
  • Default

Primary Key Contraints

  • Used as a unique indentifier for a row in a table
  • There is only one PK (Primary Key) contraint allowed on a table
  • Can have more than one unique contraints
  • PK's become FK's (Foreign Key) when creating relations among tables

Primary Key Example

When creating a table: 


              CREATE TABLE IF NOT EXISTS format
                (ID int PRIMARY KEY,
                type varchar(20),
                download boolean,
                language varchar(30),
                max_number_of_views int,
                updated_at timestamp);

When altering a table:


              ALTER TABLE format
              ADD PRIMARY KEY (ID);

Foreign Key Contraints

  • Points to a PK in another table
  • Enforces referential integrity

Foreign Key Example

When creating a table: 


              CREATE TABLE IF NOT EXISTS format
                (ID int PRIMARY KEY,
                film_id int FOREIGN KEY REFERENCES file(film_id),
                type varchar(20),
                download boolean,
                language varchar(30),
                max_number_of_views int,
                updated_at timestamp);

Foreign Key Example

When altering a table:


              ALTER TABLE format
              ADD FOREIGN KEY (film_id)
              REFERENCES film(film_id);
